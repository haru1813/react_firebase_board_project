2. Firebase Auth 세션 종료
3. 로그인 페이지로 이동		 없음

2.2 게시판 관련 유스케이스

유스케이스 ID 유스케이스명 액터 전제조건 기본 흐름 대안 흐름
UC-004 게시글 작성 인증된 사용자 로그인 상태여야 함 1. 글쓰기 버튼 클릭
2. 제목, 내용 입력
3. 유효성 검증
4. Firestore에 저장
5. 게시글 목록으로 이동		 3a. 유효성 검증 실패 시 에러 메시지 표시
UC-005 게시글 목록 조회 인증된 사용자 로그인 상태여야 함 1. 게시판 목록 페이지 접근
2. Firestore에서 게시글 목록 조회
3. 최신순 정렬
4. 목록 화면에 표시		 2a. 게시글이 없을 경우 빈 목록 표시
UC-006 게시글 상세 조회 인증된 사용자 로그인 상태여야 함 1. 게시글 클릭
2. Firestore에서 게시글 조회
3. 조회수 증가
4. 상세 정보 화면에 표시		 2a. 존재하지 않는 게시글일 경우 에러 메시지 표시
UC-007 게시글 수정 인증된 사용자 (작성자) 로그인 상태이고 게시글 작성자여야 함 1. 수정 버튼 클릭
2. 기존 내용 로드
3. 수정 내용 입력
4. 유효성 검증
5. Firestore 업데이트
6. 상세 페이지로 이동		 4a. 유효성 검증 실패 시 에러 메시지 표시
5a. 권한이 없을 경우 에러 메시지 표시
UC-008 게시글 삭제 인증된 사용자 (작성자) 로그인 상태이고 게시글 작성자여야 함 1. 삭제 버튼 클릭
2. 삭제 확인 다이얼로그 표시
3. 확인 클릭
4. 연관 댓글 삭제
5. 게시글 삭제
6. 게시글 목록으로 이동		 3a. 취소 클릭 시 다이얼로그 닫기
5a. 권한이 없을 경우 에러 메시지 표시

3. 액터 정의

액터명 설명 관련 유스케이스
비인증 사용자 로그인하지 않은 사용자 회원가입, 로그인
인증된 사용자 로그인한 사용자 로그아웃, 게시글 작성, 게시글 목록 조회, 게시글 상세 조회
게시글 작성자 게시글을 작성한 인증된 사용자 게시글 수정, 게시글 삭제

4. 관계 정의

4.1 포함 관계 (Include)

  • 인증 확인: 게시글 작성, 목록 조회, 상세 조회, 수정, 삭제 시 반드시 인증 상태를 확인해야 함

4.2 확장 관계 (Extend)

  • 현재 시스템에서는 확장 관계가 없음

5. 비기능 요구사항

성능 요구사항

  • 페이지 로딩 시간: 3초 이내
  • 게시글 목록 조회: 1초 이내
  • 실시간 데이터 동기화 지연: 1초 이내

보안 요구사항

  • 모든 게시판 기능은 인증된 사용자만 사용 가능
  • 게시글 수정/삭제는 작성자만 가능
  • Firestore 보안 규칙을 통한 데이터 접근 제어
